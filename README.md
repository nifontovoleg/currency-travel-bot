# ✈️ Travel Wallet Bot

**Миникошелёк для путешественника** — Telegram-бот на Python, который помогает вести бюджет в поездке: конвертирует расходы в домашнюю валюту, хранит баланс в двух валютах и ведёт историю трат.

> Отправьте боту сумму числом — он пересчитает её и предложит учесть как расход.

---

## 📌 О проекте

Путешествуя за границу, легко потерять контроль над тратами: цены в чужой валюте, разные курсы в обменниках, несколько поездок параллельно. **Travel Wallet Bot** решает эту задачу просто и наглядно.

Бот создаёт «кошелёк» для каждого путешествия с валютной парой (например, `RUB → CNY`), фиксирует курс обмена и показывает остаток сразу в двух валютах. Любая сумма, которую вы вводите, воспринимается как расход в валюте страны пребывания — бот конвертирует её и спрашивает, учесть ли трату.

### Возможности

| Функция | Описание |
|--------|----------|
| ✈️ **Новое путешествие** | Страна отправления → страна назначения → курс → стартовый баланс |
| 💱 **Живой курс** | Запрос курса через [api.exchangerate.host](https://api.exchangerate.host) |
| ✏️ **Ручной курс** | Можно ввести курс с обменника, если API-курс не подходит |
| 💸 **Быстрые расходы** | Введите число — бот конвертирует и предложит учесть |
| 🗺️ **Несколько поездок** | Переключение между путешествиями одним нажатием |
| 💰 **Двойной баланс** | Остаток всегда в валюте поездки и домашней валюте |
| 📋 **История** | Последние 20 расходов по активному путешествию |
| 💱 **Смена курса** | Обновление курса с пересчётом баланса |

## 🚀 Быстрый старт

### 1. Клонирование и окружение

```bash
git clone <url-репозитория>
cd currency-travel-bot

python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Настройка `.env`

Создайте файл `.env` в корне проекта:

```env
EXCHANGERATE_ACCESS_KEY=ваш_ключ_api_exchangerate_host
TELEGRAM_TOKEN=ваш_токен_от_BotFather
```

| Переменная | Где получить |
|-----------|--------------|
| `EXCHANGERATE_ACCESS_KEY` | [exchangerate.host](https://exchangerate.host) |
| `TELEGRAM_TOKEN` | [@BotFather](https://t.me/BotFather) в Telegram |

### 3. Запуск

```bash
# Windows
venv\Scripts\python.exe bot.py

# macOS / Linux
python bot.py
```

После запуска откройте бота в Telegram и отправьте `/start`.

## 💬 Как пользоваться

### Создание путешествия

1. Нажмите **✈️ Создать новое путешествие**
2. Введите страну отправления (например, `Россия`)
3. Введите страну назначения (например, `Китай`)
4. Подтвердите курс или введите свой вручную
5. Укажите стартовую сумму в домашней валюте

Пример ответа бота:

```
🎉 Путешествие создано!

📍 Валютная пара: RUB → CNY
💱 Курс: 1 RUB = 0.0882 CNY
💰 Стартовый баланс: 1 500 000.00 RUB = 132 277.50 CNY

Отправляйте суммы расходов числом — я буду конвертировать и предлагать учесть их.
```

### Учёт расходов

Просто отправьте сумму числом:

```
100
```

Бот ответит:

```
💸 100.00 CNY = 1 133.98 RUB

Учесть как расход?
[ ✅ Да ]  [ ❌ Нет ]
```

### Главное меню

| Кнопка | Действие |
|--------|----------|
| ✈️ Создать новое путешествие | Начать новую поездку |
| 🗺️ Мои путешествия | Переключиться между поездками |
| 💰 Баланс | Текущий остаток в двух валютах |
| 📋 История расходов | Список последних трат |

## 🔌 API

Все запросы курсов идут **только** на:

```
GET http://api.exchangerate.host/convert
```

Параметры: `access_key`, `from`, `to`, `amount`.

Модуль `current_api.py` обрабатывает:
- повторные попытки при `429 Too Many Requests`
- таймауты и ошибки соединения
- понятные сообщения об ошибках на русском

## ⚠️ Решение проблем

### `409 Conflict: terminated by other getUpdates request`

Запущено **несколько экземпляров** бота с одним токеном. Остановите все:

```powershell
# Windows
Get-CimInstance Win32_Process -Filter "Name = 'python.exe'" |
  Where-Object { $_.CommandLine -like '*bot.py*' } |
  ForEach-Object { Stop-Process -Id $_.ProcessId -Force }
```

Затем запустите бота **один раз**.

### `429 Too Many Requests` / лимит API

Превышен лимит запросов к exchangerate.host. Бот автоматически повторяет запрос, но на бесплатном тарифе лучше:
- не запускать скрипты с частыми запросами к API
- использовать ручной курс при создании путешествия
- не перезапускать бота слишком часто во время разработки

### Бот не отвечает

1. Проверьте, что `bot.py` запущен и в консоли нет ошибок
2. Убедитесь, что `TELEGRAM_TOKEN` в `.env` верный
3. Проверьте интернет-соединение
